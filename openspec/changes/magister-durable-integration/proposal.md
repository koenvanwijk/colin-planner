# Proposal: Durable Magister integration

## Why
Current Magister access is via a fragile login automation + local scripts. We need a more durable, repeatable integration for roster + homework with minimal breakage.

## Scope
- Prefer official/low-risk access when possible
- If unofficial API is required: make it robust and configurable
- Standardize data extraction (rooster + huiswerk) into planner pipeline

## Non-Goals
- Full-gradebook automation
- Anything that violates ToS without explicit approval
